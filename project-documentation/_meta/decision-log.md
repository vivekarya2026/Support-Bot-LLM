# Decision Log — SupportKit Voice Module

| ID | Date | Decision | Context | Alternatives | Rationale |
|----|------|----------|---------|--------------|-----------|
| ADR-001 | 2026-07-13 | VITS is the default TTS engine; XTTS-v2 is opt-in behind `TTS_ENGINE=xtts` + `COQUI_TOS_AGREED=1` | XTTS-v2 weights are CPML (non-commercial only); Coqui Inc. shut down Jan 2024, so no commercial license can ever be purchased. SupportKit is a white-label commercial platform. XTTS is also 10–30 s/reply on CPU. | (a) XTTS-only, (b) VITS-only, (c) cloud TTS | User-confirmed. Default must be commercially safe and CPU-interactive; XTTS stays available for non-commercial deployments. |
| ADR-002 | 2026-07-13 | STT via faster-whisper in the same Python sidecar; browsers never talk to the sidecar directly | Coqui TTS does no STT. App has no auth; sidecar must not become a public surface. | (a) Web Speech API in browser, (b) cloud STT, (c) separate STT service | MIT license, ~100 languages, auto-detect, decodes browser codecs via PyAV. Proxying through Next.js keeps botKey validation and flags in one place. |
| ADR-003 | 2026-07-13 | Hands-free mode is half-duplex: mic is fully disabled while TTS plays; interruption is tap-based | Echo (bot hearing itself) is the dominant failure mode of naive full-duplex; `echoCancellation:true` is not reliable across devices/embeds. | (a) full duplex + AEC, (b) voice-activity barge-in | Half-duplex is deterministic and shippable; voice-interrupt is Phase E once AEC confidence exists. |
| ADR-004 | 2026-07-13 | Voice config lives as columns on `bots` (migration v2), exposed to the widget only as booleans + language — never voice IDs | Follows the platform's established `UPDATABLE`/`toPublicConfig` plumbing; voices resolve server-side. | JSON blob column; separate table | Matches existing patterns; keeps the public config surface minimal. |
| ADR-005 | 2026-07-13 | MVP includes full hands-free conversation (stakeholder decision, against the phased recommendation) | Product owner selected "full hands-free from day one". | Push-to-talk-only MVP | Risk accepted; mitigations: half-duplex, streaming captions, spike-gated latency budgets. |
| ADR-006 | 2026-07-13 | Pin `transformers>=4.44,<5` in voice-service | coqui-tts 0.27.x imports `isin_mps_friendly`, removed in transformers 5.x; uv resolved 5.13.1 and imports failed. | Wait for upstream fix; fork | Discovered during setup; recorded in requirements.txt + setup.sh verification step. |
| ADR-007 | 2026-07-14 | Hindi TTS ships via the fairseq/MMS VITS bridge, license-badged **CC-BY-NC-4.0 (non-commercial)** in the voice catalog | Spike found no permissively-licensed Hindi VITS in the Coqui zoo; Meta MMS checkpoints are CC-BY-NC. Hindi STT (Whisper, MIT) is unaffected. | Drop Hindi TTS; XTTS-only Hindi (also non-commercial); train/license a voice | Same opt-in-with-badge pattern as XTTS: the capability exists, the license is surfaced per voice, so commercial operators can exclude it knowingly. Spike budgets confirmed: STT ~2.1 s for a 9 s clip (wav/m4a/webm identical), VITS EN ~2.0 s, HI ~0.7 s. |
